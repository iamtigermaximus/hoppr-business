# Content Creation Scenario — Hoppr Business

## The bar: *Punainen Kukko* (The Red Rooster)

A 120-capacity cocktail bar in Kallio, Helsinki. Run by **Mikko**, the owner. He's been open for 18 months, doesn't have a marketing budget, and currently posts to Instagram himself using phone photos. He just signed up for Hoppr Pro.

Mikko's goal today: create a weekend promotion, make it look professional, and get it in front of customers.

**What happens:** The system builds a structured prompt from his chip selections ("Warm & Cozy" = professional bar photography with warm amber lighting, exposed brick, candlelit atmosphere…) and runs it through a compliance filter — no underage imagery, no excessive drinking depictions, no drinking-and-driving references. The API downloads each, uploads to Cloudinary under `hoppr/bars/punainen-kukko/ai-generated/`, and returns permanent Cloudinary URLs.

## The tech behind each step

| Step | What runs |
|---|---|
| Step 3: AI understands the brief | `POST /api/auth/bar/[id]/create/suggest` → DeepSeek |
| Step 4: AI generates variants | `POST /api/auth/bar/[id]/promotions/ai-generate` → DeepSeek |
| Step 5: Structured chip selection | Client-side (AIImageGenerator component) — builds prompt from Style × Subject × Composition presets |
| Step 5: Compliance filter | `checkPromptCompliance()` — blocks if Finnish Alcohol Act patterns matched, returns warnings for borderline content |
| Step 5: AI generates image | `POST /api/auth/bar/[id]/images/generate` → FLUX.2 Klein-9b via Black Forest Labs API → Cloudinary (permanent storage) |
| Step 6: Live preview | Client-side React rendering (ConsumerPreviewPanel) |
| Step 7: Compliance check | Real-time keyword/pattern matching against Finnish Alcohol Act §50 |
| Publish: Multi-platform post | `POST /api/auth/bar/[id]/create/submit` → DB + Instagram Graph API |
| 24h later: Push notification | `scheduler` cron → FCM to devices |
| 1 week later: ROI + benchmarking | `analytics-aggregation` cron + `insights` cron |

## Provider flexibility

The image generation is provider-agnostic. Switching from FLUX to another provider is a one-line `.env` change — no code deploys needed:

```env
# Current (FLUX.2 Klein-9b, $0.015/image at 1024×1024)
AI_IMAGE_PROVIDER=flux
AI_IMAGE_API_KEY=bfl-...

# Future alternatives (same interface, different cost/quality tradeoffs)
# AI_IMAGE_PROVIDER=openai    # DALL-E 3, $0.04/image, 1 image per call
# AI_IMAGE_PROVIDER=grok      # Grok Imagine, $0.02/image, up to 10 per call
```

In development, `AI_IMAGE_PROVIDER=mock` (or unset) serves free placeholder images so the full flow can be tested without spending a cent.
